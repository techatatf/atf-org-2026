# CLAUDE.md

This file gives agent guidance for working in this repository.

## Project Direction

This repository is in a transitional state. It currently contains an African
Technology Forum prototype comparison app with selectable visual versions,
theme context, feedback controls, and version-specific section branches.

That prototype architecture is not the destination. The approved direction is a
single production ATF Website V2 implementation based on the design reference in
`docs/design-ref/`.

Future implementation work should preserve the useful foundations of the app
while removing prototype comparison concepts from the public site.

## Canonical Docs

Use these files as the source of truth for the V2 implementation:

- `docs/Design Ref Implementation/PRD.md`
- `docs/Design Ref Implementation/Goal Prompt.md`
- `docs/design-ref/ATF Website v2.html`
- `docs/design-ref/README.md`
- `docs/design-ref/colors_and_type.css`

Source precedence:

1. PRD and Goal Prompt define what to build and what obsolete prototype code to
   remove.
2. Design-reference files define visual language and interaction references.
3. Existing source code defines route structure and reusable foundations to keep
   where they do not conflict.
4. Starter-app copy and old prototype-comparison assumptions are historical.

## Commands

```bash
bun --bun run dev          # Start dev server on port 3000
bun --bun run build        # Build for production: vite build && tsc
bun --bun run test         # Run all tests with vitest
bun --bun run test <file>  # Run a single test file
```

## Foundations To Preserve

- TanStack Router file-based routing in `src/routes/`
- Shared layout boundaries for nav, route outlet, and footer
- Reusable React components and shadcn-style UI primitives
- Tailwind CSS v4 with Vite
- Existing route URLs unless a change is intentionally required
- Existing page content defaults where they still support the ATF site
- Public asset workflow under `public/`

## Transitional Code To Replace During V2 Work

- `src/lib/themes.ts`
- `src/contexts/ThemeContext.tsx`
- `src/contexts/FeedbackContext.tsx`
- `src/components/switcher/`
- URL search-param behavior for visual versions
- Version-specific section branching in `src/components/sections/`
- Visible prototype switchers, feedback panels, accent pickers, and devtools in
  the public site

Do not treat those pieces as architecture to preserve unless the PRD explicitly
keeps a smaller replacement.

## Routing

Routes are defined in `src/routes/` and the route tree is generated by TanStack
Router. The V2 reference is a single-page design artifact, but this app should
remain a multi-page route-based site.

## Styling

The existing app uses Tailwind CSS and CSS custom properties. The V2
implementation should replace the selectable theme system with static ATF design
tokens or a smaller design-system layer based on the design reference.

## Path Aliases

`@/*` maps to `./src/*` through `tsconfig.json` and `vite.config.ts`.
